# USER.md 提取规则示例 v1

这是一个示例提取 prompt，用于让裁判理解“生成 USER.md 时应该遵守哪些规则”。请把真实线上提取 prompt 粘贴到配置页并另存为新版本后使用。

## R1 事实来源

只从用户明确表达、旧 USER.md、可靠上下文中提取用户画像。不要把助手建议、系统行为、搜索结果、模型猜测当作用户事实。

## R2 长期稳定性

只记录对后续长期服务有价值的信息，例如稳定偏好、身份背景、长期目标、明确限制、持续计划。一次性请求、临时状态、当前任务流程通常不写入 USER.md。

## R3 更新逻辑

如果新信息补充旧画像，应合并到已有条目；如果新信息修正旧画像，应更新旧条目；如果新旧信息冲突且无法判断，应保守记录冲突或不更新，不要强行编造结论。

## R4 覆盖与取舍

明确、稳定、对未来有用的信息应尽量保留；弱相关、低价值、重复、噪声信息应舍弃。

## R5 边界控制

不要记录敏感隐私、短期情绪、临时查询、工具调用过程、对话格式细节、助手承诺，除非用户明确要求长期记住且符合安全边界。

## R6 表达格式

USER.md 应结构清晰、简洁、可维护。相同含义避免重复表达，条目应尽量是事实陈述，不要写评测过程或模型推理过程。

## R7 空更新

如果对话中没有值得长期记忆的信息，可以输出空 USER.md 或保持旧 USER.md，不应为了有输出而生成无依据内容。
