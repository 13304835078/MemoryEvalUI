# MemoryEvalUI 协作者快速说明

这是源码协作版，不包含打包产物、真实配置、上传文件、评测结果、闭环运行状态或 API Token。

## 1. 环境

推荐 Python 3.11。

```powershell
pip install -r requirements.txt
```

如果使用项目专用虚拟环境，先激活环境后再安装依赖。

## 2. 运行

源码方式启动：

```powershell
streamlit run app.py
```

或使用命令行菜单：

```powershell
python run.py
```

默认访问地址：

```text
http://localhost:8501
```

## 3. 配置

复制示例配置：

```powershell
copy config\local_config.example.json config\local_config.json
```

然后填写真实 API 地址、Token、模型名和需要的推理参数。`config/local_config.json` 不提交到 Git。

## 4. 当前主线功能

- `user_md_update`：USER.md 用户画像提取、绝对评测、提示词改进和闭环实验。
- `long_memory`：MEMORY.md 长期记忆提取、绝对评测、提示词改进和闭环实验。
- `day_memory`、`summary`：仍是预留任务，暂未作为主线验收。
- `人工审核结果评估`：历史 GSB 双模型对比入口，当前主线已暂时放弃 GSB，但保留页面和测试。

## 5. 主要代码位置

- `pages/`：Streamlit 页面。
- `src/schema.py`：Case、EvalResult、EvalConfig、任务类型和评分维度。
- `src/extraction/`：USER.md / MEMORY.md 提取。
- `src/loaders/`：Excel、JSONL、Markdown 数据加载。
- `src/eval/`：Judge 调用、评测流程、JSON 校验、统计和稳定性分析。
- `src/loop/`：闭环实验编排与进度状态。
- `src/ui/`：配置、文件管理、后台任务、提示词编辑、提示词改进和展示组件。
- `prompts/judge/`：裁判提示词。
- `prompts/generation/`：提取提示词，MEMORY.md 使用 YAML create/update 模板。
- `rules/`：评分规则。
- `scripts/`：命令行转换、评测、冒烟测试和打包脚本。
- `tests/`：回归测试。

## 6. 常用测试

```powershell
python -m pytest -q
python scripts/smoke_pages.py
```

提交涉及打包或页面启动时，建议额外运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build_release.ps1
```

## 7. 开发注意事项

- 不要把真实 Excel、真实结果、日志和 Token 提交到仓库。
- 修改行为时同步更新测试；修改页面文案时至少跑页面冒烟测试。
- 新增提示词不要覆盖稳定版本，另存为新文件。
- 闭环自动生成的候选提示词只作为候选版本，需要人工复核后再进入正式评测。
- MEMORY.md 提取中，评测人切换会重置旧 MEMORY.md；并发只能跨评测人片段并发，同一评测人片段内部保持顺序。
