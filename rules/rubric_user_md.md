# USER.md 更新评测规则 (Rubric v1)

## USER.md 核心职责回顾

USER.md 是用户的长期画像文件，目标是沉淀用户的**稳定特征**：
- **身份属性**：姓名、年龄、职业、常驻地、家庭角色等
- **关联人物信息**：家庭成员、关键相关人物及宠物信息
- **稳定习惯**：作息、饮食等长期重复的行为模式
- **兴趣爱好**：用户明确表达长期喜好或感兴趣的领域
- **交互偏好**：称谓、语气、篇幅、格式等对 AI 的期望

USER.md **不应**记录：
- 瞬时性信息（验证码、快递单号、临时链接）
- 瞬时动作与情绪（"正在坐地铁"、"今天心情差"）
- 单次查询与任务（查天气、一次性文案润色）
- 缺乏偏好的即时消费（"帮我点杯奶茶" vs "我喜欢喝奶茶"）
- 目标和计划（考试备考、健身计划，归 MEMORY.md）
- 需要跟踪的事项、用户关注点、车机操控习惯（归 MEMORY.md）

## 评分维度

每个维度 0-5 分，分数含义：

| 分 | 含义 |
|----|------|
| 5 | 完美，毫无瑕疵 |
| 4 | 良好，偶有轻微问题 |
| 3 | 及格，有明显但不致命的问题 |
| 2 | 较差，多个中等问题或一个严重问题 |
| 1 | 很差，严重问题为主 |
| 0 | 完全错误或不可用 |

### 维度 1：正确性 (correctness) — 权重 30%

判断新 USER.md 是否忠实于旧 USER.md 和对话记录。

- **5 分**：所有记录均来自用户明确表达，无幻觉、无事实错误
- **4 分**：1-2 处轻微不确定描述，但不影响整体准确性
- **3 分**：1 处明显的事实错误或 2-3 处轻微过度推理
- **2 分**：多处事实错误，或基于 assistant 内容做出的过度推理
- **1 分**：严重幻觉，新增了大量用户未提及的信息
- **0 分**：完全错误，与旧画像和对话记录无关

### 维度 2：完整性 (coverage) — 权重 20%

判断是否保留了应进入长期记忆的重要信息。

- **5 分**：所有关键信息均已覆盖，无遗漏
- **4 分**：遗漏了 1 条次要信息
- **3 分**：遗漏了 1 条重要信息，或 2-3 条次要信息
- **2 分**：遗漏了多条重要信息
- **1 分**：仅覆盖了不到一半的关键信息
- **0 分**：完全没有提取有效信息，或旧画像内容无故丢失

### 维度 3：更新合理性 (update_logic) — 权重 20%

判断是否正确处理旧记忆和新信息的交互。

- **5 分**：正确的新增、无冲突时保留旧记忆、冲突时正确覆盖
- **4 分**：更新基本正确，有 1 处合并或覆盖不够完美
- **3 分**：存在 1 处旧记忆被错误覆盖或 1 处该删除的旧信息未被删除
- **2 分**：存在多处更新逻辑错误
- **1 分**：旧记忆被大量错误删除或覆盖
- **0 分**：完全无视旧 USER.md，重新生成了一份无关联的新画像

### 维度 4：记忆边界 (memory_boundary) — 权重 15%

判断是否正确区分了 USER.md（画像）和 MEMORY.md（事项/计划）的边界。

- **5 分**：所有内容均属于画像范畴，无边界越界
- **4 分**：1-2 处边缘内容（可争议但总体合理）
- **3 分**：1 处明显越界（如把旅行计划、考试备考写入 USER.md）
- **2 分**：多处越界，或 1 处严重越界（如大段跟踪事项）
- **1 分**：大量短期/跟踪事项污染了画像
- **0 分**：完全混淆了画像和记忆的职责

### 维度 5：去重与凝练 (conciseness) — 权重 10%

判断输出是否简洁、去重、原子化。

- **5 分**：每条原子化表达，无重复，无冗余修饰词
- **4 分**：1 处轻微重复或可以更凝练
- **3 分**：存在 1 处明显重复，或整体略显冗长
- **2 分**：多处重复，或信息碎片化严重
- **1 分**：严重冗长或大量重复
- **0 分**：完全无法理解的格式

### 维度 6：格式合规 (format) — 权重 5%

判断输出格式是否符合要求。

- **5 分**：完全符合 `- 字段：内容` 格式，字段合法
- **4 分**：1 处格式小瑕疵
- **3 分**：多处格式问题，或包含 Think/推理内容混入输出
- **2 分**：严重格式错误，或输出不可解析
- **1 分**：完全不符合要求的格式
- **0 分**：非 Markdown、纯 JSON、或不可读格式

## 错误标签定义

| 标签 | 触发条件 |
|------|---------|
| `hallucination` | 新增了用户从未提及的信息 |
| `wrong_fact` | 用户提到的信息被错误记录（张冠李戴、数值错误等） |
| `missing_key_info` | 应该提取的关键信息被遗漏 |
| `over_memory` | 把一次性任务/临时情绪写入了长期画像 |
| `short_term_pollution` | 单次查询、瞬时信息污染了长期画像 |
| `conflict_not_resolved` | 新旧信息冲突时未正确处理 |
| `duplicate_memory` | 同一信息重复记录多次 |
| `verbose_or_noisy` | 冗长啰嗦，或包含大量无关描述 |
| `format_error` | 格式不符合要求或输出中包含 Think 推理内容 |
| `privacy_sensitive` | 敏感信息处理不当 |
| `unclear_update` | 更新意图不清晰，无法判断是新增还是覆盖 |

## Fatal Error 判定

以下情况标记为 `fatal_error=true`：
- 候选人输出是所有非法的格式（完全不可解析的乱码、结构化 JSON 而非 Markdown 等）
- 评分结果为空的 JSON 或所有维度全部 0 分
- 模型完全拒绝输出或输出明显切截的错误
